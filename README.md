# In The Moment

A native **SwiftUI** iOS app where artists and event companies post photos and
videos from their events, and fans can browse and **download** them to use however
they like.

Each creator can run **multiple independent events**; every event is its own page
that owns its own media collection.

- **Live API:** https://inthemoment-api.fly.dev
- **Repo:** https://github.com/andersonmourot/inthemoment-app

## Features

- **Discover** — a public feed of published events, with search.
- **Event pages** — cover, details, and a grid of photos/videos.
- **Full-screen viewer** — view photos and play videos, then **Save to your photo library**.
- **Creator accounts** — sign up / sign in (JWT), then create events and upload media.
- **Shared backend** — events and uploads are shared across all users via the live API.

## Architecture

Three pieces, all written in Swift:

```
.
├── Package.swift                 # SwiftPM manifest for the core library
├── Sources/InTheMomentCore/      # Shared models + logic (no UIKit/SwiftUI) — built on Linux/CI
│   ├── Models/                   # Creator, Event, MediaItem (Codable DTOs)
│   ├── Store/                    # EventStore protocol + InMemory/File/API stores + HTTPTransport
│   ├── Auth/                     # AuthClient, AuthSession, AuthenticatedTransport
│   ├── EventFeed.swift           # Pure search/sort/group helpers
│   └── SampleData.swift          # Deterministic seed data
├── Tests/InTheMomentCoreTests/   # XCTest suite (runs on Linux)
├── App/InTheMoment/              # SwiftUI app (views, AppModel, services)
│   ├── Services/                 # AuthService, Keychain, TokenHolder, MediaStorage…
│   └── Views/                    # Discover, Event, Creator, Profile, Auth
├── Server/                       # Vapor backend (Fluent + SQLite) — reuses InTheMomentCore
├── project.yml                   # XcodeGen project definition (source of truth)
├── Dockerfile / fly.toml         # Container build + Fly.io deploy config
└── InTheMoment.xcodeproj         # GENERATED by XcodeGen — do not hand-edit
```

The **server and the app share the exact same `InTheMomentCore` models**, so the
JSON contract can never drift between client and backend.

### Swappable backend

The app talks only to the [`EventStore`](Sources/InTheMomentCore/Store/EventStore.swift)
protocol, so the backend is a one-line injection into `AppModel`. Three implementations ship:

| Store | Use |
| --- | --- |
| `InMemoryEventStore` | previews & tests |
| `FileEventStore` | on-device JSON persistence |
| `APIEventStore` | REST backend — **the app default**, pointed at the live server |

`APIEventStore` takes an injectable `HTTPTransport`. The app wraps it in
[`AuthenticatedTransport`](Sources/InTheMomentCore/Auth/AuthenticatedTransport.swift)
to attach the signed-in creator's bearer token to every mutating request.

### Auth

- Register/login returns a **JWT** + the creator profile (`AuthSession`).
- The token is stored in the **Keychain** (`TokenHolder`) and restored on launch.
- **Read** routes are public; **write** routes (`POST/PUT/DELETE` events & media)
  require a token and enforce that you can only modify your own events.

## Requirements

- **App:** Xcode 15+ (iOS 16.0 deployment target), macOS
- **Core / Server:** Swift 6.0.3 (no Xcode needed)
- **Deploy:** [flyctl](https://fly.io/docs/flyctl/) + a Fly.io account

## Open & run the iOS app

```bash
xcodegen generate          # regenerate the project after any file/layout change
open InTheMoment.xcodeproj # run the "InTheMoment" scheme on an iOS 16+ simulator
```

> The app defaults to the live API. To point it at a local server, set the
> `ITM_API_BASE_URL` environment variable in the scheme (see `.env.example`).

## Build & test the core (no Xcode)

```bash
swift build
swift test
```

## Run the server locally

```bash
cd Server
swift run InTheMomentServer serve --hostname 127.0.0.1 --port 8080
# DATABASE_PATH and JWT_SECRET are read from the environment (see .env.example)
```

## Deploy the server

```bash
flyctl deploy --remote-only --app inthemoment-api
```
